# 快速开始 - 完整版

## 5 分钟快速体验 SQL2Bot

### 前置准备

确保已安装：
- Java 17+
- Maven 3.6+
- MySQL 8.0+
- Node.js 16+
- Redis 6.0+ (可选)

### 第一步：启动后端

```bash
# 1. 进入项目目录
cd /c/tecdo2/sql2bot

# 2. 配置数据库（如果还没有）
# 编辑 src/main/resources/application.properties
# 修改数据库连接信息

# 3. 启动应用
./mvnw spring-boot:run
```

后端将在 `http://localhost:8082` 启动

### 第二步：启动前端

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖（首次运行）
npm install

# 3. 启动开发服务器
npm run dev
```

前端将在 `http://localhost:3000` 启动

### 第三步：配置数据源

1. 打开浏览器访问 `http://localhost:3000`
2. 点击左侧菜单"数据源管理"
3. 点击右上角"新建数据源"按钮
4. 填写表单：
   ```
   名称: 测试数据库
   类型: MySQL
   主机: localhost
   端口: 3306
   数据库名: your_database
   用户名: root
   密码: your_password
   ```
5. 点击"测试连接"确保连接成功
6. 点击"确定"保存

### 第四步：创建语义模型

#### 方式一：自动导入（推荐）

1. 在数据源列表中，找到刚创建的数据源
2. 点击"发现表"按钮
3. 系统会自动读取所有表结构
4. 选择要导入的表
5. 点击"导入"，系统会自动创建模型和字段定义

#### 方式二：手动创建

1. 点击左侧菜单"表模型管理"
2. 选择数据源
3. 点击"新建模型"
4. 填写表信息：
   ```
   数据源: 选择刚创建的数据源
   物理表名: users
   显示名称: 用户表
   描述: 存储用户基本信息
   主键: id
   可见性: 开启
   ```
5. 保存后，点击"字段"按钮
6. 为每个字段添加定义：
   ```
   字段名: user_id
   显示名称: 用户ID
   描述: 用户唯一标识
   数据类型: BIGINT
   字段类型: 维度
   可空: 否
   ```

### 第五步：配置表关系

1. 点击左侧菜单"表关系配置"
2. 点击"新建关系"
3. 配置关系：
   ```
   源表: 订单表
   目标表: 用户表
   关系类型: 一对多
   JOIN 条件:
     - 源表字段: user_id
     - 目标表字段: id
   描述: 订单属于用户
   ```
4. 点击"确定"保存

### 第六步：配置大模型

1. 点击左侧菜单"大模型配置"
2. 填写 API 配置：
   ```
   Base URL: https://api.anthropic.com
   API Key: 你的 API Key
   模型: claude-3-5-sonnet-20241022
   Max Tokens: 4096
   Temperature: 0.7
   ```
3. 可选：启用向量 RAG
   ```
   启用向量 RAG: 开启
   Top-K: 5
   ```
4. 点击"保存配置"

### 第七步：开始查询

1. 点击左侧菜单"查询"
2. 在输入框中输入自然语言问题：
   ```
   有多少个用户？
   ```
3. 点击"发送"或按 Enter
4. 系统会：
   - 理解你的问题
   - 生成 SQL 查询
   - 执行查询
   - 返回结果和解释

5. 尝试更复杂的查询：
   ```
   上个月注册的用户有多少？
   哪个用户的订单最多？
   各类别商品的销售情况如何？
   ```

6. 支持多轮对话：
   ```
   第一轮: 有多少个用户？
   第二轮: 他们的平均订单数是多少？
   第三轮: 再看看上个月的数据
   ```

## 常见问题

### Q1: 后端启动失败
**检查**:
- 数据库是否正在运行
- 数据库连接信息是否正确
- 端口 8082 是否被占用

### Q2: 前端无法连接后端
**检查**:
- 后端是否已启动
- 前端代理配置是否正确（vite.config.js）
- 浏览器控制台是否有错误

### Q3: 查询返回错误
**检查**:
- 语义模型是否完整
- 表关系是否正确配置
- LLM API 配置是否正确
- 查看后端日志获取详细错误

### Q4: SQL 生成不准确
**优化**:
- 为表和字段添加详细描述
- 正确标记字段类型（维度/度量）
- 配置完整的表关系
- 启用向量 RAG

## 技巧和最佳实践

### 1. 语义建模技巧
- 使用清晰的业务术语作为显示名称
- 为每个字段添加详细的业务描述
- 正确区分维度和度量
- 配置完整的表关系

### 2. 查询技巧
- 使用简单明确的问题
- 避免模糊的表述
- 利用多轮对话细化问题
- 查看生成的 SQL 学习

### 3. 性能优化
- 只导入必要的表
- 隐藏不需要查询的表
- 为常用字段添加索引
- 启用查询缓存（规划中）

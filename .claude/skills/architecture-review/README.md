# 架构审查技能

> Java 项目结构、包和依赖方向的宏观级分析

## 功能

从高层次分析项目架构：
- 包组织（按层 vs 按功能 vs 六边形）
- 层之间的依赖方向
- 模块边界和耦合
- 架构反模式（上帝包、贫血领域等）

## 何时使用

- "审查这个项目的架构"
- "这个包结构好吗？"
- "检查我们是否遵循整洁架构"
- "查找架构违规"
- 重大重构工作之前

## 关键概念

### 包策略

| 策略 | 最适合 | 权衡 |
|----------|-------------|--------------|
| 按层 | 小项目，快速启动 | 分散相关代码 |
| 按功能 | 中型项目，清晰模块 | 需要共享内核 |
| 六边形 | 复杂领域，可测试性 | 更多仪式感 |

### 依赖方向

```
外层（框架） → 适配器 → 应用 → 领域（内层）

规则：依赖仅指向内部
```

## 使用示例

```
你：审查这个项目的架构

Claude：[分析包结构]
        [检查依赖方向]
        [识别违规]
        [提供优先级建议]
```

## 检查内容

1. **包结构** - 组织、命名一致性
2. **依赖方向** - 领域隔离、无框架泄漏
3. **层边界** - 适当的关注点分离
4. **模块边界** - 清晰的 API、封装
5. **可扩展性** - 功能能否被提取？

## 相关技能

- `solid-principles` - 类级设计（此技能是包/模块级）
- `design-patterns` - 实现模式（此技能是结构性）
- `clean-code` - 代码质量（此技能是架构质量）

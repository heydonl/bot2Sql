## 🚀 如何使用

### 1. 安装依赖
```bash
cd frontend
npm install
```

### 2. 启动开发服务器
```bash
npm run dev
```

### 3. 访问应用
打开浏览器访问：http://localhost:3000

### 4. 功能演示

**添加数据源**:
1. 访问"数据源管理"
2. 点击"添加数据源"
3. 填写连接信息
4. 测试连接
5. 保存

**执行查询**:
1. 访问"查询测试"
2. 输入问题："Count datasources"
3. 点击"执行查询"
4. 查看结果

**配置大模型**:
1. 访问"大模型配置"
2. 设置 API 信息
3. 调整参数
4. 保存配置

## 🐛 已知问题

1. **大模型配置保存**
   - 问题：配置保存功能未连接后端 API
   - 影响：配置无法持久化
   - 解决：需要后端提供配置保存 API

2. **查询历史持久化**
   - 问题：查询历史只保存在内存中
   - 影响：刷新页面后历史丢失
   - 解决：使用 localStorage 或后端存储

3. **表模型管理未实现**
   - 问题：只有占位页面
   - 影响：无法通过界面管理表模型
   - 解决：开发完整的表模型管理功能

## 💡 使用建议

### 开发环境
1. 确保 Node.js >= 16
2. 使用 npm 或 yarn 安装依赖
3. 启动后端服务（8082 端口）
4. 启动前端服务（3000 端口）

### 生产部署
1. 运行 `npm run build` 构建
2. 将 `dist` 目录部署到 Web 服务器
3. 配置 Nginx 反向代理后端 API

### 开发建议
1. 使用 Vue DevTools 调试
2. 查看浏览器控制台的网络请求
3. 参考 Element Plus 文档
4. 遵循 Vue 3 Composition API 规范
